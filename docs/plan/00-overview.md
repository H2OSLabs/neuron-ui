# neuron-ui 项目总体规划

## 三大目标及其关系

```
目标 3: 设计规范开发 ──────► 目标 1: AI 可读可用 ──────► 目标 2: 拖拉拽页面搭建
    (地基)                      (骨架)                      (上层应用)
```

### 依赖关系

| 目标 | 依赖 | 说明 |
|------|------|------|
| 目标 3: 设计规范开发 | 无 | **地基**，所有组件的开发基础 |
| 目标 1: AI 可读可用 | 目标 3 | 需要组件已按规范开发，才能生成结构化元数据 |
| 目标 2: 拖拉拽搭建 | 目标 1 + 目标 3 | 需要组件渲染能力（目标 3）和组件 Schema（目标 1） |

### 建议执行顺序

```
阶段 1 — 基建 (目标 3)
├── 设计 Token 包
├── 基础组件开发 (按设计规范)
└── Storybook 组件文档

阶段 2 — AI 能力 (目标 1)
├── 组件元数据 Schema
├── 组件清单 Manifest
└── AI 使用协议 & Prompt 模板

阶段 3 — 可视化搭建 (目标 2)
├── 页面 Schema 格式
├── 拖拉拽编辑器
├── 属性编辑面板
└── 预览与发布
```

### 三个目标的共享产出

| 产出物 | 目标 3 | 目标 1 | 目标 2 |
|--------|--------|--------|--------|
| Design Tokens 包 | ✅ 开发依据 | ✅ AI 约束参考 | ✅ 编辑器配色面板 |
| TypeScript 类型定义 | ✅ 开发规范 | ✅ AI 理解 Props | ✅ 属性编辑面板生成 |
| 组件 Schema (JSON) | — | ✅ AI 读取 | ✅ 拖拽组件注册 |
| 组合规则 | — | ✅ AI 正确嵌套 | ✅ 拖拽约束 |
| 渲染引擎 | ✅ 组件实现 | — | ✅ 画布渲染 |
